Mas por que Semântica nos elementos do HTML5?

Com HTML4, os desenvolvedores usavam seus próprios nomes de atributos para os elementos da página de estilos:

header, top,bottom, footer, menu, nav, main, container, content, artcle, sidebar, topnav, ...

Isso tornou impossível para os robôs de busca identificar o conteúdo correto das páginas web.

Com elementos HTML5 como: < header> < footer> < nav> < section> < article>, isso se torna mais fácil.

De acordo com o W3C, a Web Semântica:

"Permite que os dados sejam compartilhados e reutilizados em aplicações, empresas e comunidades."

• < section>: seção do documento;

• < nav>: links de navegação;

• < aside>: define um conteúdo sem relevância para o contexto da página. Ou seja, se ele for removido o sentido da página não perde valor(ex: biografias,avisos etc);

• < header>: cabeçalho de uma página ou seção;

• < footer>: rodapé de uma página ou seção:ex:Normalmente um rodapé contém o autor das informações do documento, copyright, links para termos de uso, informações de contato, etc.

• < main>: define o conteúdo principal da página. Tem apenas um main em toda a página.

• < article>: conteúdo independente da página;

Video


• Utilização da tag video com definição das fontes de mídia;
• As possíveis fontes e seus respectivos tipos de dados são encapsulados em uma tag source;
• Alguns dos principais atributos:
• Controls: adiciona os principais controle de mídia no player;
• Autoplay: assim que a página é carregada o vídeo já começa a ser tocado;

Codigo:

<video width="400" controls> <source src="https://www.w3schools.com/htmL/mov_bbb.mp4" type="video/mp4"> <source src="https://www.w3schools.com/htmL/mov_bbb.mp4" type="video/ogg"> Your browser does not support HTML5 video. </video> <p> Video courtesy of <a href="https://www.bigbuckbunny.org/" target="_blank">Big Buck Bunny</a>. </p>

Resultado:

Video courtesy of Big Buck Bunny.
