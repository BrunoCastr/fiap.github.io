Eventos


Eventos são ações que acontecem a elementos html. Quando usamos javascript em nossas páginas podemos determinar o que acontecerá nesses eventos. Utilizamos estes eventos como atributos que contém código javascript para disparar as ações.

<button class="button2" onclick='alert("Olá, sou uma caixa de alerta!");'>Button</button>

Resultado:




Através dos eventos podemos disparar ações no próprio elemento ou em qualquer outro da página.

<button class="button2" onclick="document.getElementById('idTitulo').innerHTML='( x_x) Formatado'">Button</button>

Resultado:

( •_•) Formate seu Texto






Obs. Para interagirmos com o próprio elemento podemos utilizar o termo “this”.

<button class="button2" onclick="this.innerHTML='(⌐■_■)'">( •_•)</button>

Resultado:

    Validação


    Uma das maiores funcionalidades do javascript são as validações sobre tipo e formato de dados, que podemos fazer diretamente do navegador, ficando assim o processo muito mais rápido. Podemos fazer validações simplesmente através de condiçõesou utilizar expressões regulares. Como o javascript roda diretamente no navegador, não devemos validar dados de usuário ou do sistema, mas apenas se os dados foram inseridos e se foram inseridos de forma correta.

    if (valor < 0 || valor > 9000 || isNaN(valor)){ alert("Este valor está muito alto!") }else{ alert("Vamos mandar seu pedido para análise!") }

    Perceba que o valor não pode ser negativo, não pode ser maior que 9000 e tem que ser um número. isNaN( ) = é uma função que verifica se seu conteúdo não é um número.

    Expressões regulares

    Especificadores


    Especificam o conjunto de caracteres a casar em uma posição.

    Descrição Tag
    Qualquer caractere, exceto a quebra de linha \n (ver flag_dotall) .
    Qualquer caractere incluído no conjunto [...]
    Qualquer caractere não incluído no conjunto [^...]
    O mesmo que [0-9] \d
    O mesmo que [^0-9] \D
    Espaço, quebra de linha, tabs etc.; o mesmo que [ \t\n\r\f\v] \s
    O mesmo que [^ \t\n\r\f\v] \S
    O mesmo que [a-za-z0-9_] (mas pode incluir caracteres unicode; ver flag_unicode) \w
    O complemento de \w \W
    Anula o significado especial do metacaractere seguinte; por exemplo, \. Representa apenas um ponto, e não o curinga \


    Quantificadores


    Definem o número permitido repetições da expressão regular precedente.

    Descrição Tag
    Exatamente n ocorrências {n}
    No mínimo n ocorrências e no máximo m {n,m}
    No mínimo n ocorrências {n,}
    No máximo n ocorrências {,n}
    0 ou 1 ocorrência; o mesmo que {,1} ?
    1 ou mais ocorrência; o mesmo que {1,} +
    0 ou mais ocorrência *
    Modera qualquer um dos quantificadores acima «q»?


    Âncoras


    Estabelecem posições de referência para o casamento do restante da regex. Note que estes metacaracteres não casam com caracteres no texto, mas sim com posições antes, depois ou entre os caracteres.

    Descrição Tag
    Início do texto, ou de uma linha com o flag re.Multiline ^
    Início do texto \A
    Fim do texto, ou de uma linha com o flag re.Multiline; não captura o \n no fim do texto ou da linha $
    Fim do texto \Z
    Posição de borda, logo antes do início de uma palavra, ou logo depois do seu término; o mesmo que a posição entre \w e \w ou vice-versa \b
    Posição de não-borda \B


    Agrupamento


    Definem ou grupos ou alternativas.

    Descrição Tag
    Define um grupo, para efeito de aplicação de quantificador, alternativa ou de posterir extração ou re-uso (...)
    Alternativa; casa a regex à direita ou à esquerda ...|...
    Recupera o texto casado no n-ésimo grupo \«n»
